# OWASP Juice Shop Security Assessment

## Overview

This repository contains my cybersecurity internship work focused on security testing and analysis of the OWASP Juice Shop application. The project involved identifying web application vulnerabilities, reviewing security controls, implementing basic logging, and documenting security findings.

## Objectives

- Perform penetration testing on OWASP Juice Shop.
- Identify common web application vulnerabilities.
- Review existing security mechanisms.
- Implement basic security logging.
- Document findings and recommendations.

## Tools Used

- OWASP Juice Shop
- OWASP ZAP
- Browser Developer Tools
- Nmap
- Node.js
- npm
- Winston
- Helmet
- Validator
- bcrypt
- JSON Web Token (JWT)
- Git & GitHub

## Activities Performed

### Week 1
- Installed and configured OWASP Juice Shop.
- Performed vulnerability assessment using OWASP ZAP.
- Identified Cross-Site Scripting (XSS) and SQL Injection vulnerabilities.
- Generated a security assessment report.

### Week 2
- Reviewed existing security controls.
- Studied Helmet for HTTP security headers.
- Reviewed Validator for input validation.
- Reviewed bcrypt for password hashing.
- Reviewed JWT authentication implementation.
- Documented observations and recommendations.

### Week 3
- Conducted additional browser-based penetration testing.
- Verified XSS and SQL Injection vulnerabilities.
- Installed and configured Winston logging.
- Reviewed HTTPS implementation as a security best practice.
- Prepared the final security report.

## Vulnerabilities Identified

- Cross-Site Scripting (XSS)
- SQL Injection

## Security Controls Reviewed

- Helmet
- Validator
- bcrypt
- JSON Web Token (JWT)
- Winston Logging
- HTTPS (Best Practice)

## Recommendations

- Validate and sanitize all user inputs.
- Use parameterized SQL queries.
- Encode output to prevent XSS.
- Enable HTTPS in production.
- Secure passwords using bcrypt.
- Maintain application logs using Winston.
- Conduct regular vulnerability assessments.
